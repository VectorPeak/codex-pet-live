# Security Policy / 安全策略

## 报告安全问题

如果发现可能泄露用户隐私、API key、聊天记录、本地文件路径、配置文件内容或发布包安全的问题，请优先使用 GitHub Security Advisories 私下报告。

如果不确定是否属于安全问题，也可以创建 issue，但请先脱敏：不要公开粘贴 API key、token、账号信息、完整隐私日志、私人聊天内容或可定位个人设备的信息。

## 脱敏建议

- 将 API key、token、cookie 替换为 `[REDACTED]`
- 将本机用户名、绝对路径、聊天内容和配置中的隐私字段删除或改写
- 只保留复现问题所需的最小日志片段
- 截图前检查系统托盘、设置页、控制台和文件路径是否包含隐私信息

## 支持范围

项目以当前公开仓库和最新 Release 为主要支持范围。第三方角色素材、私有 LLM 服务、个人改包版本或未公开插件中的问题，请在报告中说明来源和改动范围。
